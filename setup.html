Step 1: Spotify API Setup

Spotify provides free API access for music search and playback previews.

  1. Go to Spotify Developer Dashboard
  2. Log in or create an account
  3. Click "Create App"
  4. Fill in:
    • App Name: TuneTogether
    • App Description: Music streaming with party mode
    • Redirect URI: http://localhost:8080
  5. Agree to terms and click "Create"
  6. Copy your Client ID and Client Secret
  7. Open config.js and paste them in the appropriate fields

Step 2: Firebase Setup (For Party Mode)

Firebase provides real-time database for synchronizing party rooms.

  1. Go to Firebase Console
  2. Click "Add Project"
  3. Name it TuneTogether and follow the wizard
  4. In your project dashboard, click the Web icon (</>)
  5. Register your app with nickname TuneTogether
  6. Copy the Firebase configuration object
  7. Paste it into config.js in the FIREBASE_CONFIG section
  8. Go to Build ‚Üí Realtime Database
  9. Click "Create Database"
  10. Choose "Start in test mode" (for development)
‚ö†Ô∏è Important: Test mode allows open access. For production, set up proper security rules!

Step 3: Local Testing

Test the application on your computer before deploying.

Using Python:

# Python 3
python -m http.server 8080

# Or Python 2
python -m SimpleHTTPServer 8080

Using Node.js:

npx http-server -p 8080

Using VS Code:

  1. Install the "Live Server" extension
  2. Right-click index.html
  3. Select "Open with Live Server"

Then visit http://localhost:8080 in your browser.

Step 4: GitHub Repository Setup

Create a repository and push your code.

  1. Go to GitHub and create a new repository
  2. Name it TuneTogether
  3. Keep it public (required for GitHub Pages)
  4. Don't initialize with README (we already have files)

In your project folder, run:

git init
git add .
git commit -m "Initial commit: TuneTogether app"
git branch -M main
git remote add origin https://github.com/ogarsh/TuneTogether.git
git push -u origin main

Step 5: Deploy to GitHub Pages

Make your app live on the internet!

  1. Go to your repository on GitHub
  2. Click Settings tab
  3. Scroll to Pages in the left sidebar
  4. Under Source, select main branch
  5. Click Save
  6. Wait 2-5 minutes for deployment
  7. Your app will be live at: https://ogarsh.tech/TuneTogether/
üéâ Congratulations! Your app is now live!

Step 6: Update API Settings

Add your GitHub Pages URL to API settings.

For Spotify:

  1. Return to Spotify Dashboard
  2. Click your app
  3. Click Edit Settings
  4. Add https://ogarsh.tech/TuneTogether/ to Redirect URIs
  5. Click Save

For Firebase:

  1. Go to Firebase Console
  2. Select your project
  3. Go to Authentication ‚Üí Settings ‚Üí Authorized Domains
  4. Add ogarsh.tech to authorized domains

üÜò Troubleshooting

Music Search Not Working

  • Check if Spotify credentials are correct in config.js
  • Ensure you're not using placeholder values
  • Check browser console (F12) for error messages
  • The app will use demo tracks if APIs aren't configured

Party Mode Not Syncing

  • Verify Firebase configuration in config.js
  • Ensure Realtime Database is enabled
  • Check database rules (should allow read/write in test mode)
  • Make sure both users are online

GitHub Pages Not Loading

  • Wait 5-10 minutes after enabling GitHub Pages
  • Check that repository is public
  • Verify all files are committed and pushed
  • Check Settings ‚Üí Pages for deployment status
